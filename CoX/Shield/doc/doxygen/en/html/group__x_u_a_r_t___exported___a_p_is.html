void xUARTConfigSet ( unsigned long  ulBase,
unsigned long  ulBaud,
unsigned long  ulConfig 
)

Sets the configuration of a UART.

Parameters
ulBaseis the base address of the UART port.
ulBaudis the desired baud rate.
ulConfigis the data format for the port (number of data bits, number of stop bits, and parity).

This function configures the UART for operation in the specified data format. The baud rate is provided in the ulBaud parameter and the data format in the ulConfig parameter.

The ulConfig parameter is the logical OR of three values: the number of data bits, the number of stop bits, and the parity. UART_CONFIG_WLEN_8, xUART_CONFIG_WLEN_7, xUART_CONFIG_WLEN_6, and xUART_CONFIG_WLEN_5 select from eight to five data bits per byte (respectively). xUART_CONFIG_STOP_1 and xUART_CONFIG_STOP_2 select one or two stop bits (respectively). xUART_CONFIG_PAR_NONE, xUART_CONFIG_PAR_EVEN, xUART_CONFIG_PAR_ODD, xUART_CONFIG_PAR_ONE, and xUART_CONFIG_PAR_ZERO select the parity mode (no parity bit, even parity bit, odd parity bit, parity bit always one, and parity bit always zero, respectively).

Returns
None.

Referenced by SensorShieldInit().

void xUARTCharGetNonBlocking ( unsigned long  ulBase)

Receives a character from the specified port.

Parameters

Gets a character from the receive FIFO for the specified port.

This function replaces the original UARTCharNonBlockingGet() API and performs the same actions. A macro is provided in uart.h to map the original API to this API.

)

Sends a character to the specified port.

Parameters
ulBaseis the base address of the UART port.
ucDatais the character to be transmitted.

Writes the character ucData to the transmit FIFO for the specified port. This function does not block, so if there is no space available, then a false is returned, and the application must retry the function later.

This function replaces the original UARTCharNonBlockingPut() API and performs the same actions. A macro is provided in uart.h to map the original API to this API.

Returns
Returns true if the character was successfully placed in the transmit FIFO or false if there was no space available in the transmit FIFO.

void xUARTBusy ( unsigned long  ulBase)

Determines whether the UART transmitter is busy or not.

Parameters
ulBaseis the base address of the UART port.

Allows the caller to determine whether all transmitted bytes have cleared the transmitter hardware. If false is returned, the transmit FIFO is empty and all bits of the last transmitted character, including all stop bits, have left the hardware shift register.

Returns
Returns true if the UART is transmitting or false if all transmissions are complete.
void xUARTIntEnable ( unsigned long  ulBase,
unsigned long  ulIntFlags 
)

Enables individual UART interrupt sources.

Parameters
ulBaseis the base address of the UART port.
ulIntFlagsis the bit mask of the interrupt sources to be enabled.

Enables the indicated UART interrupt sources. Only the sources that are enabled can be reflected to the processor interrupt; disabled sources have no effect on the processor.

The ulIntFlags parameter is the logical OR of any of the following:

  • xUART_INT_ERROR - All Error interrupt
  • xUART_INT_RT - Receive Timeout interrupt
  • xUART_INT_TX - Transmit interrupt
  • xUART_INT_RX - Receive interrupt
Returns
None.

void xUARTRxErrorGet ( unsigned long  ulBase)

Gets current receiver errors.

Parameters
ulBaseis the base address of the UART port.

This function returns the current state of each of the 4 receiver error sources. The returned errors are equivalent to the four error bits returned via the previous call to UARTCharGet() or UARTCharGetNonBlocking() with the exception that the overrun error is set immediately the overrun occurs rather than when a character is next read.

Returns
Returns a logical OR combination of the receiver error flags, xUART_RXERROR_FRAMING, xUART_RXERROR_PARITY, xUART_RXERROR_BREAK and xUART_RXERROR_OVERRUN.
void xUARTRxErrorClear ( unsigned long  ulBase)

Clears all reported receiver errors.

Parameters
ulBaseis the base address of the UART port.

This function is used to clear all receiver error conditions reported via UARTRxErrorGet(). If using the overrun, framing error, parity error or break interrupts, this function must be called after clearing the interrupt to ensure that later errors of the same type trigger another interrupt.

Returns
None.

void xUARTFlowControlSet ( unsigned long  ulBase,
unsigned long  ulMode 
)

Sets the UART hardware flow control mode to be used.

Parameters
ulBaseis the base address of the UART port.
ulModeindicates the flow control modes to be used. This is a logical OR combination of values xUART_FLOWCONTROL_TX and xUART_FLOWCONTROL_RX to enable hardware transmit (CTS) and receive (RTS) flow control or xUART_FLOWCONTROL_NONE to disable hardware flow control.

Sets the required hardware flow control modes. If ulMode contains flag xUART_FLOWCONTROL_TX, data is only transmitted if the incoming CTS signal is asserted. If ulMode contains flag xUART_FLOWCONTROL_RX, the RTS output is controlled by the hardware and is asserted only when there is space available in the receive FIFO. If no hardware flow control is required, xUART_FLOWCONTROL_NONE should be passed.

Note
The availability of hardware flow control varies with the Stellaris part and UART in use. Please consult the datasheet for the part you are using to determine whether this support is available.
Returns
None.

Enables SIR (IrDA) mode on the specified UART.

Parameters
ulBaseis the base address of the UART port.
ulBaudis the desired baud rate.
ulConfigis the data format for the port (number of data bits, number of stop bits, and parity).
ulModeis the IrDA mode.

This function configures the UART IrDA for operation in the specified data format. The baud rate is provided in the ulBaud parameter, the data format in the ulConfig parameter, and the ulMode is provided in the ulMode parameter.

The ulConfig parameter is the logical OR of three values: the number of data bits, the number of stop bits, and the parity. xUART_CONFIG_WLEN_8, xUART_CONFIG_WLEN_7, xUART_CONFIG_WLEN_6, and xUART_CONFIG_WLEN_5 select from eight to five data bits per byte (respectively). UART_CONFIG_STOP_1 and xUART_CONFIG_STOP_2 select one or two stop bits (respectively). xUART_CONFIG_PAR_NONE, xUART_CONFIG_PAR_EVEN, xUART_CONFIG_PAR_ODD, xUART_CONFIG_PAR_ONE, and xUART_CONFIG_PAR_ZERO select the parity mode (no parity bit, even parity bit, odd parity bit, parity bit always one, and parity bit always zero, respectively).

The ulMode parameter can be values:

  • xUART_IRDA_MODE_NORMAL - IrDA normal mode
Note
SIR (IrDA) operation is not supported on Sandstorm-class devices.
Returns
None.
